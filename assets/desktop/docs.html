Ambiente Pipeline / Curral
Contexto

Módulo

animal – Criar / editar animal

Usado em animal_create e animal_update.
Não usa get_modulos para prefill

O formulário de animal é sempre exibido dentro do pipeline ao clicar em “Criar animal”. Ele não é controlado por get_modulos. As keys abaixo servem para payloads de sync e para alinhar os nomes de atributos entre front e backend.

get_modulos – Exemplo

Exemplo de como enviar as abas de movimentação:

{
  "modulo": "movimentacao",
  "abas": [
    {
      "titulo": "lotes",
      "campos": [
        { "key": "lote", "value": "", "type": "OS" }
      ]
    },
    {
      "titulo": "pastos",
      "campos": [
        { "key": "pasto", "value": "", "type": "OS" }
      ]
    },
    {
      "titulo": "fazendas",
      "campos": [
        { "key": "fazenda_destino", "value": "", "type": "OS" },
        { "key": "lote", "value": "", "type": "OS" },
        { "key": "pasto", "value": "", "type": "OS" }
      ]
    }
  ]
}

O front compara o titulo da aba em lowercase com lotes, pastos e fazendas para aplicar os valores.

get_modulos – Exemplo completo

{
  "modulo": "saida_animais",
  "abas": [
    { "titulo": "Venda", "campos": [ /* ver tabela acima */ ] },
    { "titulo": "Empréstimo", "campos": [ /* proprietario_destino, ... */ ] },
    { "titulo": "Doação", "campos": [ /* idem Empréstimo */ ] },
    { "titulo": "Ajuste inventário", "campos": [ /* idem Empréstimo */ ] },
    { "titulo": "Morte", "campos": [ /* causa_morte, ... */ ] }
  ]
}

O backend pode omitir abas que não deseja usar ou enviar campos: []; o front completa com as keys padrão de DEFAULT_ABA_CAMPOS_KEYS.

Resumo rápido

Mapeamentos e operações

Para implementar o backend com segurança.

FORM_FIELD_IDS (get_modulos)

  • movimentacao/lotes: lote → pipelineMovLoteDestino
  • movimentacao/pastos: pasto → pipelineMovPastoDestino
  • movimentacao/fazendas: fazenda_destino, lote, pasto
  • pesagem/pesagem: colaborador, tipo_pesagem, peso, data_pesagem
  • saida_animais/venda: todas as keys financeiras/GTA e destino
  • saida_animais/emprestimo, doacao, ajuste inventário: layout enxuto (5 campos)
  • saida_animais/morte: causa, responsavel, datas, imagem, movimentacao_saida_animal

Operações offline (OFFLINE_OPS)

  • animal_create, animal_update, animal_create_peso
  • create_saida_animais_venda, ..._morte, ..._emprestimo, ..._aju_inventario, ..._doacao
  • movimentacao_entre_lotes, movimentacao_entre_pastos, movimentacao_entre_fazendas

O tipo de saída exibido na UI (movimentacao_saida_animal) é convertido para a operação correta via SAIDA_TIPO_TO_OFFLINE_OP em config.js.
